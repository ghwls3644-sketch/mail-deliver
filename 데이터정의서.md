## 3. 엔티티 클래스

### 3-1. `Building(Entity)` (`game/map.py`)

배달 가능한 건물 오브젝트 (소포·등기 배달 대상).

| 속성 | 타입 | 설명 |
|------|------|------|
| `building_id` | `str` | 유일한 건물 ID — `Mail.target_id`와 매핑 |
| `label` | `str` | 화면에 표시되는 건물 이름 |
| `interactable` | `bool` | 항상 `True` (E키 감지 대상) |
| `interact_label` | `str` | 플레이어 힌트에 표시되는 문자열 |

#### 인스턴스 파라미터

| 파라미터 | 타입 | 기본값 | 설명 |
|---------|------|--------|------|
| `building_id` | `str` | — | 건물 고유 ID |
| `label` | `str` | — | 표시 이름 |
| `pos` | `tuple[float,float,float]` | — | 월드 좌표 `(x, y, z)` |
| `model_path` | `str \| None` | `None` → `'cube'` | GLB 모델 경로 |
| `col` | `Color` | `color.white` | 모델 없을 때 사용 색상 |
| `scale` | `tuple` | `(2, 2, 2)` | 오브젝트 스케일 |

#### `interact(player)` 동작

1. `player.mails`에서 `target_id == building_id`이고 미배달인 우편물 검색
2. 없으면 "배달할 우편물이 없어요" 메시지
3. `mail_type == 'express'`면 NPC 전달 안내 메시지 (완료 처리 안 함)
4. 그 외 → `mail.delivered = True`, 건물 초록 점멸 후 복원

| 속성 | 타입 | 설명 |

| `interact_label` | `str` | `'우체통에 편지 넣기'` |

#### 인스턴스 파라미터

| 파라미터 | 타입 | 기본값 | 설명 |
|---------|------|--------|------|
| `building_id` | `str` | — | 대응 건물 ID |
| `pos` | `tuple` | — | 월드 좌표 |
| `box_color` | `Color` | `color.red` | 우체통 색상 |

#### `interact(player)` 동작

1. `player.mails`에서 `target_id == building_id`, `mail_type == 'letter'`, 미배달인 우편물 검색
2. 없으면 "넣을 편지가 없어요"
3. 있으면 → `mail.delivered = True`, 초록 점멸 후 원래 색 복원

### 3-3. `NPC(Entity)` (`game/npc.py`)

주민 NPC — 대화 및 관계 시스템.

| 속성 | 타입 | 기본값 | 설명 |
|------|------|--------|------|
| `npc_id` | `str` | — | NPC 고유 ID |
| `name` | `str` | — | 표시 이름 |
| `building_id` | `str` | — | `Mail.target_id`와 연결 (등기 전달 매핑) |

| `interactable` | `bool` | `True` | E키 감지 대상 |
| `interact_label` | `str` | `'{name}에게 말 걸기'` | 힌트 표시 문자열 |

#### 인스턴스 파라미터

| 파라미터 | 타입 | 기본값 | 설명 |
|---------|------|--------|------|
| `npc_id` | `str` | — | NPC 고유 ID |
| `name` | `str` | — | 표시 이름 |
| `building_id` | `str` | — | 연결 건물 ID |
| `pos` | `tuple` | — | 월드 좌표 |
| `col` | `Color` | `color.cyan` | NPC 몸통 색상 |

#### `interact(player)` 동작 우선순위

1. **등기 우편물 있으면** → `mail.delivered = True`, 관계 +1 시도, 서명 메시지
2. **일반 대화** → `_DIALOGUES[relation]` 순환 출력
   - 관계 단계 0일 때 3회 대화 후 자동으로 단계 1 승격

### 3-4. `Player(Entity)` (`game/player.py`)

플레이어 캐릭터 — 이동 및 상호작용.

| 속성 | 타입 | 초기값 | 설명 |
|------|------|--------|------|
| `speed` | `float` | `5` | 이동 속도 (units/sec) |
| `mails` | `list[Mail]` | — | 현재 소지 우편물 목록 |
| `interactables` | `list[Entity]` | `[]` | 상호작용 가능 오브젝트 목록 |
| `_interact_cooldown` | `float` | `0` | E키 쿨다운 (초) |
| `hint` | `Text` | — | 화면 하단 E키 힌트 텍스트 |

#### 이동 방식

- WASD 입력을 카메라 yaw 기준으로 변환 (아이소메트릭 쿼터뷰 대응)
- x축 / z축 분리 이동 → 벽 슬라이딩 지원

#### 상호작용 조건

- 가장 가까운 `interactable=True` 오브젝트가 거리 `2.5` 이내
- E키 입력 시 `nearest.interact(self)` 호출
- 쿨다운: `2.0`초

## 5. 맵 파일 포맷 (`game/maps/*.json`)

에디터(`editor.py`)가 저장하고, 나중에 게임(`game/map.py`)이 읽어야 할 공유 스펙.

### 최상위 구조

| 키 | 타입 | 설명 |
|----|------|------|
| `objects` | `list[object]` | 배치된 오브젝트 목록 |

### 오브젝트 항목

| 필드 | 타입 | 설명 | 예시 |
|------|------|------|------|
| `name` | `str` | 팔레트 표시 이름 | `"tree_oak"` |
| `model` | `str` | GLB/GLTF 경로 (프로젝트 루트 기준) | `"assets/kenney_nature-kit/Models/GLTF format/tree_oak.glb"` |
| `pos` | `[float, float, float]` | 월드 좌표 `[x, y, z]` | `[2.0, 0, 4.0]` |
| `scale` | `[float, float, float]` | 스케일 (게임 기준 — `SC` 프리셋 값) | `[1.5, 1.5, 1.5]` |
| `rot_y` | `float` | Y축 회전 (도) | `90` |

> **주의**: `scale`은 에디터 표시 배율(`SC_DISPLAY_MULT`)이 아닌 게임용 원본 스케일(`SC`)을 저장.
> 에디터가 크게 보여주는 것은 표시 전용이며 JSON에는 반영되지 않음.

### 예시

```json
{
  "objects": [
    {
      "name": "straight",
      "model": "assets/kenney_city-kit-roads/Models/GLB format/road-straight.glb",
      "pos": [0.0, 0, 0.0],
      "scale": [2, 2, 2],
      "rot_y": 0
    },
    {
      "name": "tree_oak",
      "model": "assets/kenney_nature-kit/Models/GLTF format/tree_oak.glb",
      "pos": [3.0, 0, 2.0],
      "scale": [1.5, 1.5, 1.5],
      "rot_y": 90
    }
  ]
}
```
