## 1. 게임 한 줄 소개

> 작은 마을의 우편배달부가 되어 편지를 나르고, 주민들과 조금씩 가까워지는 잔잔한 3D 쿼터뷰 배달 게임

---

## 2. 장르

- **메인**: 힐링 시뮬레이션
- **서브**: 생활 + 가벼운 시간관리

### 핵심 감성 방향

| ❌ 피해야 할 것 | ✅ 추구하는 것 |
|---------------|--------------|
| 압박감 있는 시간 제한 | 느긋한 하루 루틴 |
| 장애물 회피 액션 | 잔잔한 동네 산책 |
| 퍼즐 풀기 긴장감 | 편지 읽는 따뜻함 |
| 점수 / 경쟁 중심 | 관계와 성장 중심 |

> **기본 철학**: 실패 페널티보다 잘 했을 때의 보상에 집중.
> 못 배달해도 게임 오버 없음 — 다음 날 이어서 배달.

---

## 3. 핵심 플레이

하루 흐름:

```
아침 출근
    ↓
우편물 수령 & 확인
(오늘 누구한테 뭐가 왔는지 훑어보기)
    ↓
동네를 걸으며 배달
(경치 구경, 주민 인사, 우편함에 넣기)
    ↓
배달 중 소소한 이벤트
(주민이 말 걸기, 편지 내용 힌트 등)
    ↓
저녁 귀환 → 하루 일기 한 줄
```

---

## 4. 플레이어 목표

- **오늘의 배달** — 다 배달하면 좋지만, 못 해도 괜찮음
- **주민 관계** — 자주 배달할수록 친해지는 사람들
- **동네 탐험** — 새로운 골목, 숨겨진 공간 발견
- **편지 스토리** — 배달하다 보면 주민들 사이의 관계가 드러남

게임오버 없음. 주민 관계는 강요되지 않음 — 배달하다 보면 생기는 것.

---

## 5. 주요 시스템

### 5-1. 배달 시스템

#### 우편물 종류

| 색상 | 종류 | 전달 방식 |
|------|------|----------|
| 🟡 노란색 | 편지 | 우체통에 투입 |
| 🟤 갈색 | 소포 | 현관 앞에 놓기 |
| 🔵 파란색 | 등기 | 주민 직접 만나서 전달 |

#### 주소 표기 방식

우편물에 **수신인 이름 + 시각적 단서** 항상 같이 표기 (맵이 바뀌어도 유지)

```
예) 김할머니 / 골목 파란 우체통 집
    이씨 / 맨션 3층 301호
```

#### 난이도 상승 방식 — 단서 제거가 아닌 환경 복잡도 증가

| 맵 단계 | 배달처 유형 | 난이도 |
|--------|------------|--------|
| 초반 마을 | 단독주택, 농가 | 집마다 외관이 달라 한눈에 구분 |
| 중반 마을 | 주택 + 소규모 상가 | 비슷한 집들 사이에서 단서로 찾기 |
| 이후 맵 | 맨션, 오피스텔 (소규모) | 건물은 찾기 쉽지만 호수까지 확인 필요 |

#### 오배달 / 지각 처리

- **오배달**: 주민이 웃으며 "이거 저 집 거 아닌가요?" 하는 식으로 자연스럽게 처리
- **지각 배달**: 패널티 없음 — 주민이 반갑게 받아줌

---

### 5-2. 시간 시스템

- 하루 제한 시간 있지만 압박감 없음
- 시간이 지나면 자연스럽게 저녁이 되고 복귀 → 다음 날 시작
- 못 한 배달은 내일 우편물에 추가됨

---

### 5-3. 주민 관계 시스템

배달할 때마다 주민과 짧은 대화.
호감도가 쌓이면 주민이 먼저 말을 걸거나 소소한 선물을 줌.
각 주민마다 조금씩 사연이 공개됨 (편지 내용 힌트 등).

```
[낯선 사람] → [아는 사이] → [단골] → [친구]
```

#### 호감도 상승 요인
- 배달 성공
- 대화 선택지 (공감하는 답변)
- 특별한 날(생일 등) 배달

---

### 5-4. 날씨 시스템 (분위기용)

맑음 / 흐림 / 비 / 눈 — **이동에 영향 없음**

- 비 오는 날: 우비 착용
- 눈 오는 날: 발자국 효과

시각적 연출만. 페널티 없음.

---

## 6. 게임 진행 구조

```
[첫날] 튜토리얼 — 3~5집, 아주 쉬운 배달
     ↓
[일과 루프] ← 게임의 중심
  아침: 우편물 수령
  낮:  배달 + (자연스럽게) 주민 대화
  저녁: 복귀 → 오늘 하루 일기 한 줄
     ↓
  [서브] 호감도 올라가면 주민이 먼저 말 걸거나 작은 선물
  [서브] 편지 내용 힌트로 주민 사연이 조금씩 보임
     ↓
[엔딩] 일정 날짜가 지나면 자연스럽게 마무리
       (모든 주민 친구 만들기는 선택 사항)
```

---

## 7. 차별점

- **실패 없는 설계**: 못해도 괜찮고, 틀려도 이야기가 이어짐
- **편지가 연결하는 스토리**: 배달하다 보면 주민들 사이의 관계가 드러남
- **조용한 루틴의 재미**: 특별한 사건 없어도 매일 반복되는 배달 자체가 힐링

## 8. 맵 구성

### 기본 개념

- 전체가 하나의 동네 — 구역 간 이동 제한 없음, 어디든 자유롭게 걷기 가능
- **해금 = 배달 담당 구역이 늘어나는 것** — 물리적 차단 없음
- 미해금 구역은 지도에서 그림자(어둡게) 처리, 실제 공간은 존재하지만 배달 대상 없음

### 구역 배치

```
┌────────────────────────────────┐
│  [구역 C]         [구역 D]     │
│  외곽 농가        해안가        │
│                                │
│       [구역 A]   [구역 B]      │
│       마을 광장  주택가 골목    │
│       🏤 우체국               │
└────────────────────────────────┘
```

구역끼리 붙어 있는 하나의 마을. 경계는 지도 표시용.
**구역 A가 중심** — 우체국 위치.

### 구역 상세

| 구역 | 이름 | 건물 | 주민 수 |
|------|------|------|--------|
| A | 마을 광장 | 우체국, 빵집, 꽃집, 집 4~5채 | 6~8명 |
| B | 주택가 골목 | 집 8~10채, 작은 공원 | 10~12명 |
| C | 외곽 농가 | 농가 3~4채, 헛간, 텃밭 | 5~6명 |
| D | 해안가 | 등대, 어부 집 2~3채, 카페 | 4~5명 |

### 해금 방식

- **해금 전**: 지도에서 그림자 표시, 공간은 존재하지만 우편물 없음
- **해금 시**: 우체국 아저씨 한마디 → 지도에서 해당 구역 선명해짐
- **순서**: A (시작) → B (며칠 후 자동) → C (B 배달 어느 정도 후) → D (선택, 후반)
